2. Realitas Ekosistem Keamanan Siber

PARADIGMA KEAMANAN: Dalam infrastruktur teknologi informasi modern, tidak ada sistem yang memiliki jaminan kebal 100%. Keamanan siber merupakan dinamika berkelanjutan (arms race) antara arsitek proteksi dan pihak pengeksploitasi.

Bahkan entitas berskala global dengan infrastruktur masif (seperti korporasi teknologi terkemuka atau lembaga pemerintahan) tetap menghadapi risiko penetrasi dari peretas berdedikasi tinggi.

VEX v1.0 direkayasa secara presisi untuk meminimalisir, menyulitkan, dan memblokir eksploitasi massal guna menjaga integritas ekonomi digital proyek Anda. VEX berfungsi sebagai lapisan mitigasi proaktif, bukan solusi absolut yang meniadakan ancaman eksploitasi tingkat lanjut. Memahami realitas ini adalah prasyarat profesionalisme dalam merilis produk berbasis teknologi.

Bagian I: Panduan Teknis (User Manual)

1. Memulai (Getting Started)

1.1 Arsitektur VEX
VEX menggunakan arsitektur Hybrid Cloud Security yang menghubungkan Engine Roblox dengan Backend Security API dan Database Terenkripsi. Filosofi kami berpusat pada mitigasi berbasis cloud yang menawarkan tingkat isolasi dan keandalan jauh melampaui sistem skrip lokal tradisional.

1.2 Alur Instalasi Paket (.rbxm)

  1. Bergabung ke server Discord resmi kami. Gunakan perintah /create <PlaceId> untuk menerbitkan Secret Key kriptografis Anda.
  2. Unduh berkas distribusi VEX_AntiCheat_v2.rbxm dari saluran resmi yang ditunjuk.
  3. Buka Roblox Studio, navigasikan ke jendela Explorer, klik kanan lalu pilih Insert from File...
  4. Distribusikan modul ke dalam hierarki layanan berikut:
    • VEX (Folder) ➡️ ServerScriptService
    • VEX_TopbarIcon ➡️ ReplicatedStorage
    • VEX_AdminPanel ➡️ StarterGui
    • VEX_ClientHandler & VEX_ExecutorScanner ➡️ StarterPlayerScripts

1.3 Aktivasi & Konfigurasi Mandiri

Prosedur Esensial: Anda wajib mengaktifkan Allow HTTP Requests melalui menu Game Settings > Security di Roblox Studio.

Konfigurasikan lisensi dan akses administrator pada berkas VEX_Config.lua:

Config.SECRET_KEY = "VEX-XXXX-XXXX"
Config.ADMIN_IDS  = { 12387945654, 12345678 }
Protokol Integritas: Berkas VEX_BackendURL.lua telah dilindungi melalui enkripsi obfuscation tingkat lanjut. Dilarang membuka, mengubah, atau mendekompilasi modul ini, karena tindakan tersebut akan merusak rute kriptografis menuju server Backend.

2. Sistem Deteksi Logika (Core Security Logic)

  • 2.1 Lingkungan Pemindaian Klien (Client-Side VM Scanner): Mengidentifikasi injeksi eksternal dan perkakas manipulasi memori melalui inspeksi berkala pada fungsi global tingkat rendah (getgenv, hookfunction, checkcaller).
  • 2.2 Analisis Perilaku Server (Server Behavioral Radar): Melakukan validasi anomali kecepatan dan elevasi melalui pemantauan koordinat fisika server (AssemblyLinearVelocity) untuk mengeliminasi manipulasi posisi lintas-klien (anti-spoof).
  • 2.3 Jaringan Pemblokiran Global (VEX-VAC Network): Sinkronisasi data pelanggaran antar instansi (Pelanggaran terverifikasi di Area A akan mereplikasi penangguhan ke Area B secara terotomatisasi).
  • 2.4 Proteksi Aset (Poison Pill Mechanism): Protokol penguncian integritas aset melalui tanda tangan digital. Jika ruang lingkup direplikasi secara paksa (contoh: SaveInstance), sistem akan memicu degradasi tata letak aset dan mentransmisikan sinyal anomali ke webhook administrator.
